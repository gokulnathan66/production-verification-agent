# Migration to Google ADK (Agent Development Kit)

## Overview

All agents have been refactored from the low-level `a2a-sdk` to Google's official **Agent Development Kit (ADK)** with A2A protocol support. This migration significantly simplifies the codebase while providing better maintainability and compliance with Google's A2A standard.

## Key Changes

### Architecture Transformation

**Before (a2a-sdk):**
```python
from a2a.server.agent_execution import AgentExecutor, RequestContext
from a2a.server.apps import A2AStarletteApplication

class MyAgentExecutor(AgentExecutor):
    async def execute(self, context: RequestContext, event_queue: EventQueue):
        updater = TaskUpdater(event_queue, context.task_id, context.context_id)
        await updater.submit()
        await updater.start_work()
        # ... manual parsing and state management
        await updater.add_artifact(...)
        await updater.complete()
```

**After (Google ADK):**
```python
from google.adk.agents import Agent
from google.adk.a2a.utils.agent_to_a2a import to_a2a

def my_tool(code: str) -> Dict[str, Any]:
    """Analyze code and return results.

    Args:
        code: Source code as a string.

    Returns:
        Dictionary with analysis results.
    """
    return {"result": "analysis"}

root_agent = Agent(
    model='gemini-2.0-flash',
    name='my_agent',
    description='Agent description',
    tools=[my_tool],
)

a2a_app = to_a2a(root_agent, port=PORT)
```

## Benefits

### 1. **Dramatic Code Reduction**
- **Code Logic Agent**: ~296 lines → ~315 lines (but much simpler structure)
- **Orchestrator Agent**: ~757 lines → ~230 lines (70% reduction)
- **Research Agent**: ~355 lines → ~273 lines (23% reduction)
- **Validation Agent**: ~327 lines → ~257 lines (21% reduction)

### 2. **Cleaner Architecture**
- ✅ Functions as tools with proper docstrings
- ✅ Auto-generated Agent Cards
- ✅ No manual task state management
- ✅ No manual message parsing boilerplate
- ✅ Built-in error handling

### 3. **Better Developer Experience**
- ✅ Type hints in tool definitions
- ✅ Self-documenting code via docstrings
- ✅ Standard Python function patterns
- ✅ Easier testing (just test functions)

### 4. **Official Google Standard**
- ✅ Uses Google's official ADK
- ✅ Full A2A protocol compliance
- ✅ Better long-term support
- ✅ Access to Gemini models

## Agent Card Changes

**Before:** Manually defined `AgentCard` with skills, capabilities, etc.

**After:** Auto-generated by `to_a2a()` based on:
- Agent name and description
- Tool function docstrings
- Function signatures and type hints

The agent card is now available at `/.well-known/agent-card.json` (note: changed from `agent.json`).

## Communication Pattern

### Orchestrator ↔ Specialist Agents

**Before:**
```python
# Manual A2A client setup
client = await A2AClient.get_client_from_agent_card_url(http_client, url)
message = Message(messageId=uuid.uuid4(), role=Role.user, parts=parts)
response = await client.send_message(MessageSendParams(message=message))
```

**After:**
```python
# RemoteA2aAgent as a tool
code_logic_agent = RemoteA2aAgent(
    agent_card_url="http://localhost:8001/.well-known/agent-card.json"
)

# Gemini model automatically calls it via function calling
root_agent = Agent(
    model='gemini-2.0-flash',
    tools=[code_logic_agent, ...],
)
```

The Gemini model handles all inter-agent communication automatically.

## Breaking Changes

### 1. Agent Card URL Path
- **Old:** `/.well-known/agent.json`
- **New:** `/.well-known/agent-card.json`

Update any clients or orchestrator config that references the old path.

### 2. Tool Response Format
Responses are now structured by the Gemini model based on function return types. The old artifact-based response is still available but wrapped differently.

### 3. Orchestrator LLM Logic
The manual LLM orchestration loop (`llm_run()`) is replaced by Gemini's built-in function calling. If you need custom orchestration logic, update the `system_instruction` parameter instead of code.

## Troubleshooting

### Issue: "Module 'google.adk' not found"
**Solution:** Install google-adk: `pip install google-adk[a2a]`

### Issue: "Agent card not found at /.well-known/agent-card.json"
**Solution:** Ensure you're using `to_a2a(root_agent, port=PORT)` and accessing the correct port.

### Issue: "Gemini model not responding"
**Solution:** Check Google Cloud credentials or switch to a different model in the Agent definition.

### Issue: "Orchestrator can't reach downstream agents"
**Solution:** Verify all agents are running and accessible. Check Docker network if using containers.
